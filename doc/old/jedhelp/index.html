Version 0.93 beta
Additions
"         F9 Scale function is enhanced to allow scaling along X,Y,Z or Grid Z axis individually, not just in all directions.
"         Holding Alt while cycling through possible items under cursor ( i.e. - when you click the left mouse button several times without moving the mouse) now cycles backwards.
"         You can now change the CMP used as palette when you view a 3DO in Resource Picker.
"         JED now shows a circle designating radius for a selected light.
"         JED now invokes an "Add COG" dialog when you click on a COG's variable of type "COG" that equals -1.
"         Added an option to hide things/lights when not in thing/light mode.
"         Holding "Alt" while dragging objects now limits its transformation to one grid axis.
"         You can now manipulate shapes (prefabs) - import them from sectors and delete them.
"         The last open file in the recent file list now moves on top
"         Pressing + and - to adjust gamma in 3D Preview in OpenGL and Direct3D IM mode doesn't display warning message about having to reload 3D preview, instead reloads it. Note that it can take some time though.
"         Added "Blocks Light" adjoin flag that tells JED to treat the adjoin as an opaque surface in lighting claculation.

Bugfixes/Improvements
"         Fixed a bug in extracting .COG from GOB/GOOs when you try to edit a .COG that's in the game's GOB/GOO files.
"         Fixed a bug in .SurfaceUpdate() DLL interface function.
"         Fixed a bug of not being able to enter capital S in layer name
"         Fixed bug in .PickResource() plugin function

Version 0.92 beta
Additions
"         When you click "Edit COG" in Placed COG editor on a COG that's inside a GOB file, JED now offers you to extract it to project directory to edit it.
"         When you preview a multiple MAT in Resource Picker, a scrollbar will now appear that will let you scroll through all cells in the file.
"         Now you can use rotate,flip and scale F9 tools on things, lights and frames as well as sectors.
"         By pressing Shift+I in 3D Preview you can now insert a thing at camera position.
"         "Join" function now works on sectors as well as surfaces.
"         Added a lot of DLL functionality
"         Template Creator now reloads template file before editing, making sure that you don't lose any changes you've made indepenedently to the .TPL file.
"         Added "Duplicate" button to Placed COG editor that lets you place another copy of the currently selected COG. Useful for doors and such. Also, Placed cog editor screen updates are sped up somewhat.
"         When you change a template in Template Creator all the things that use it are now updated.
"         The plugin interface unit is now the same for Delphi 2.0 and 3.0

Bugfixes/Improvements
"         Fixed an omission in DLL plug-in interface that made it impossible to make DLLs using some programming languages.
"         Fixed a bug causing plug-ins written in Delphi 3.0 cause access violation on exit from JED.

Version 0.91 beta
Additions
"         Added an option to make Placed COGs window stay on top.
"         Added DLL Plugin interface (old OLE still works).

Bugfixes/Improvements
"         Template Creator now allows you to replace existing templates (be careful when doing that).
"         Changed 3DO preview slightly so that its failure to initialize doesn't disable template/3DO pickers.
"         Fixed bug causing 3D Preview go black when you look straight up or straight down.
"         "Multiple" now appears after width/height info for multiple MAT files in Resource Picker.

Version 0.9 beta
Additions
"         Added overlapping sector detection in consistency checker. Disabled by default (as it slows things down), you can enable it in "Options|Misc. Options".
"         Added an option to rotate/flip around Grid in Tools.
"         Added a "Snap View to Object" (Alt+S) option - centers the view on the selected object.
"         Added "Join" function - position two surfaces back to back, select them both and press "J" - they will be cleaved and their common area will become a pair of adjoined surfaces.
"         Added "Extrude And Expand" feature (Ctrl+X).
"         Added "Bring thing/light To Surface" function. Select a surface, then switch to thing mode, select a thing (or to light mode and select a light) and press "B". The thing/light will be brought to the middle of the surface and offset to the appropriate distance from it (0.01 units for lights and half of thing's bounding box for things).
"         Added "Insert thing at surface" function to 3D Preview. Select a surface in 3D preview and press "I" - a thing will be placed at the middle of it.
"         Added and option to confirm "Revert to Saved" function.
"         Added "Layer" field in "Find Surface" dialog.
"         Added "Prep" button in Tools rotation dialog. The way it works is you select two surfaces (multiselect one and select another) and press this button, JED calculates the rotation parameters for a rotation that would place these two surfaces in one plane. After this preparation you only need to select the sectors to rotate and press "Rotate".
"         Added "Jump to Object" function that lets you jump to a specific object (sector, surface,etc) by its number. You can also jump to surfaces by their absolute number (their number in JKL) - you just need to enter one number instead of two.

Bugfixes/Improvements
"         Consistency checker now doesn't report "Sector not convex" error for sectors with "Show as 3DO" flag set - i.e. the sectors that are to be exported as 3DOs, not part of level geometry.
"         Redone 3D Preview low level code.
"         Fixed a bug in undo not undoing sector deletion completely (surfaces adjoined to the deleted sectors weren't restored).
"         Fixed a bug of undo information not being saved for some Tools operations.
"         Fixed a bug that made small surfaces color-filled when using texturing tools.
"         Redone lighting calculation (sped up about 4 times). Kept an option to revert to the old lighting calculation for compatibility.
"         You can now add/subtract things and surfaces to/from multiselection in 3D Preview by Shift+Mouse Click. It is an equivalent of selecting this objects and pressing space. I.e. - it is affected by "Multiselection mode" setting.
"         When you click on surface/thing in 3D preview, the main view is no longer centered on the selected object. Use "Snap view to Object" if you want to center it.
"         Fixed a bug causing "Invalid pointer operation" when performing operation on things with large (10 and more) number of frames.
"         JED now prints a warning when you form non-convex or non planar surfaces and non-convex sectors by merging.
"         3DO export and 3DO import are changed slightly:
         Export:
         - The meshes are now named by the layer names.
         - Sectors in the same layer are merged into one mesh.
         - Adjoined surfaces are not exported.
         Import:
         - Each mesh is imported to a layer named like the mesh.
         - "Preview as 3DO" flag is set on imported sectors.

OLE Scripting
"         Added .IsConvex function to TOLESector. Returns true if sector is convex and false otherwise.
"         Added .IsConvex and .IsPlanar functions to TOLESurface. Returns true if sector is convex/planar and false otherwise.
"         Added a bunch of OLE functions to retrieve consistency errors.

K3D Preview Window



The 3D Preview window is one of the most invaluable editing tools in JED. By this window, you can test how your level looks like without having to start Jedi Knight over and over just to check out some texturing changes. Besides, reasonable texturing and stitching is only possible in the 3D Preview.

You navigate in the window like used in Jedi Knight: using arrow keys. With PgUp / PgDn you can watch downwards and upwards; with A and Z you get higher or lower... see the keyboard summary for the complete set of keys.

To select things or faces in 3D Preview, you just click on them with the mouse. Pressing Enter then will pop-up the thing or face's Item Editor, and double-clicking will allow you to change the texture for the surface or the template for the thing.

To stitch textures, you have to notice two key combinations: Ctrl+Ins and Shift+Ins. Ctrl+Ins will select a starting surface for stitching. By selecting a neighboring surface and pressing Shift+Ins, you will stitch the texture to this surface, so that the texturing is continuous.

You also can move textures manually on the surface. By pressing Shift+arrow keys, you shall scroll the texture around. By Ctrl+left/right, you shall rotate the texture. And by pressing Alt+, or Alt+. you can scale the texture up or down. The settings for these operations are specified in the Map Settings.

A very useful feature for setting up ghost things or cameras is the function Copy Camera as Frame called by the F key. This will copy the current 3D Preview camera setting (X/Y/Z;Pitch/Yaw/Roll) as a frame you can paste into a thing's properties afterwards.

Find Dialog

In JED, you are able to search for items (these are: sectors, faces and things) by defining the search criteria more or less close. Here the Find Sector dialog:



The criteria differ in the different modes and are derived from the Item Editor settings.
The selection of the criteria is pretty simple. Each criterion has a combo box which contains the hit operators and an edit field which contains the data to which these operators relate. So, for example, the operator "Is not" 0 in the Flags criterion (see picture) means, that JED will find every sector in the level which Flags field is not equal 0.
The operators may vary depending on the related field type. There are following operators:

Is       all items with the property set to the specified value will be found
Is not   all items with the property not set to the specified value will be found
<        all items with the property (e.g. number) bigger than the specified value will be found
>        all items with the property (e.g. number) smaller than the specified value will be found
Set      (for flags) all items with the property having the specified value bit(s) set, will be found
Not set  (for flags) all items with the property having the specified value bit(s) not set, will be found
=        same as Is
<>       same as Is not
Contains         all items with the property containing the specified value as a part will be found
Doesn't contain  all items with the property not containing the specified value as a part will be found
' '      an empty field deactivates this search criterion

Some of the criteria have a selection "..." button which enables to select the value from the list. For example, the Flags, or the resource criteria like textures, sounds or colormaps.

The Query Results field specifies the action performed after the search. The options are:

Added to multiselection will multiselect all the items with all the matching parameters. Useful if you e.g. want to replace one certain texture everywhere in the level or remove one certain flag.

Subtracted from mutiselection will subtract the matching items from an existing multiselection. This one is useful if you e.g. want to change parameters of all sectors in the level except the underwater ones.

Focused in the editor will select the first found item. You can select the next by selecting "Find Again" from the Edit Menu.

Introduction


         JED is a Jedi Knight: Dark Forces II level editor. With it, you are able to create custom Single player (SP) and Multi player (MP) levels. However, to get used to it, you have to learn its multiple operations and possibilities. That's what this help file is for. Contrary to the previous help files for JED (up to 0.51), there will not only be keyboard summaries and What's New? topics. The help contains information on the menus, dialog boxes etc.

JED is a level editor for Jedi Knight/MotS. Its main purpose is to edit the geometry aspect of the level. Although JK levels consist of many aspects which can be edited, JED does not edit textures, sound, or text resources editing like COG, AI etc. JED can link them with the level and treat them properly, but the main things JED does are geometry, object placement, COG linking, texturing. For other editing aspects, there are other programs like CogWriter, MatMaster, or TrajJedi, just to name a few.

3DO
#3DO stands for "3D object", a three-dimensional model. In Jedi Knight, all powerups, weapons and enemies/persons are made as textured 3D polygon models. Also pieces of architecture can be built of 3DO objects.

Adjoin
#Adjoin connects two sectors together so that you can see/get from one into another sector. Imagine a room sector and a door track sector which have to be connected. To connect them, they have to have two surfaces in common (of equal shape/location). These you can adjoin, thus creating a link between two sectors/surfaces.

AI
#AI files are text file definitions of the Artificial Intelligence of things. AI files contain so-called instincts which declare which abilities e.g. an enemy has and how they are set.

Cleave
#Cleave means cut a sector/surface/edge into 2 parts. For example, if you cleave a sector (Press C, and draw a cleaving line), it shall be divided into 2 adjoined sectors. Same with surfaces and edges. The opposite of cleaving is Merging.

Code Alliance
#The Code Alliance is a group of hobby programmers which origined from Dark Forces level development. The Code Alliance makes utilities/programs for DF, JK, Outlaws, XvT editing.
The homepage of the Code Alliance: http://www.darkjedi.com/

COG
#COG is the JK/MotS internal C-like language which handles the level's dynamics: COG controls the goal completion, weapons, force powers, special behavor of persons/items....

Container
#"Containers" are usually large data files which contain multiple resource files. For example, in JK, the whole mass of texture, sound, 3DO etc. resources is contained in a few large GOB files.
Custom levels should also usually be distributed in GOB/GOO containers to avoid conflicts and simply because one big file is easier to handle than 100 small ones.

Convex
#Convex is a condition for a surface/sector to be to be valid in Jedi Knight. Convex means that there are no "cut-ins". Let's suppose, you take a surface and draw endless lines on each edge of it. If no line crosses an edge, the surface is convex. If one does, the surface is concave and not valid. Same is valid for sectors.

Dark Forces
#Dark Forces (DF) was the prequel game to Jedi Knight. It is a solely SP game about Kyle Katarn for which a large bunch of excellent Single Player levels exist. The Code Alliance has its beginning in the Dark Forces editing community.

You can import the geometry and objects of Dark Forces levels (.LEV) into JED by the File->Import command.

DFU
#DFU stands for "Dark Forces Unit". One DFU is 8 texels (texture pixels). A DFU approximately corresponds to 25 cm; thus, one JKU is equal 40 DFU.

Edge
# A surface is built out of edges which are built by two vertices.

GOB
#GOB is the container type Jedi Knight uses (and also used to be in Dark Forces). MotS uses .GOB files with the .GOO ending.

To GOB a project means to create a container file with all the project resources ready for distribution.


GOO
#GOO is a container file used in MotS. There is no internal difference between GOO and GOB files.

See GOB definition.

JK
#JK stands for "Jedi Knight", the award-winning Dark Forces sequel. JED is the level editor for Jedi Knight.

JKU
#JKU means "Jedi Knight Unit" and is a unit used to measure the distances in. 1 JKU corresponds to 10 meters, and to 320 texels (texture pixels) at scaling 1.

KEY
#KEY files are trajectory files for 3D objects and meshes (which are subparts of 3D objects). KEY files can control flying ships, but also the movements of the joints of the persons, or of weapons.

Layer
#Layers are JED-internal structures to order the level and make it more comfortable to edit. You can define different layers (e.g. Floor1, Floor2 etc.), assign sectors, things, lights etc. to certain layers and then turn them on/off for easier editing.

Light
#Lights are JED-internal light source objects. You set them up in your sectors to make them more or less lighted. You can change the light's intensity and range, thus enabling you to create either diffuse and equal lighting or dim and mysterious one. The calculation of the level's lighting is performed from the Tools menu.

MotS also supports colored lights.

MAT
#MAT files are texture files used by Jedi Knight. MAT files can contain multiple cels (=frames); they also can contain up to 4 MipMaps (texture detail levels).
There are also "Color MATs" which actually consist only of solid colors and are used e.g. for particle systems.

MotS
#MotS is an abbreviation for "Mysteries Of The Sith" addon CD. MotS editing has some enhanced aspects like colored lighting. Also, MotS has some more goodies on CD (like more atmospheric 3DOs, more enemies, and more sounds).

Mysteries Of The Sith
#Mysteries Of The Sith is an addon CD for Jedi Knight. MotS editing has some enhanced aspects like colored lighting. Also, MotS has some more goodies on CD (like more atmospheric 3DOs, more enemies, and more sounds).

Plugin
#A Plugin is an external Program or DLL which corresponds with JED, thus adding to JED's functionality. A plugin can get some JED data (multiselection, statistics, Resource picking etc.), and write its own data after processing (for example, dublicate items in some way, or make some operations automatically etc.). You can get JED plugins on the Code Alliance homepage.

PUP
#PUP files are Puppet definitions of 3D objects. They assign KEY files to different actions an object can have. The PUP definition regulates the movements of a 3D object.

Sector
# The sectors are kind of "rooms". In JK, levels are not built in space, but kind of are carved out of solid matter. Sectors are rooms where the player can move. The action can only happen in the sectors. Sectors must be convex in Jedi Knight.

Shape
#Shapes are predefined sector forms which can be created by pressing the K key + moving the mouse. There are basic simple shapes already available in JED. However you can define your own shapes by exporting sector(s).

Stitch
#To stitch a texture means to continue the texture alignment on further walls. For example, you have a room with walls built of multiple surfaces. You only have to setup the texturing for one wall, and then use stitching to make the texturing "flow" from one surface to another.

Surface
# Surfaces are the planar ingredients of a sector. Surfaces are e.g. walls, floors, ceilings. They must be planar and they must be convex.

Template
#Things are based on templates. The templates are inheritable structures of properties, i.e. one template can inherit most properties from another one. In the templates, many properties of things are declared, e.g. the 3D model, the AI logic, the health, the damage, the Thingflags.

Thing
#"Things" are objects which do not belong to geometry, i.e. are theoretically able to move. Most things are shown as 3D models, called 3DOs but there are also invisible things ("ghosts"), sprite things (explosions, bubbles) or particle structures. Things are based on templates.

Thing Frame
#The Frames stand for different locations of a thing. They are needed for COG programming of moving objects. For example, to enable an elevator to move from one stop to another, the elevator thing will require 2 frames which stand for the elevator's starting and ending position.

Vertex
# Vertices are points of the polygons which lie on the corners. The polygon is "spanned" on the vertices.

WAV
#WAV is the common sound format used by Windows PCs. JK also uses this format (up to 22,5 kHz; 16 Bit) for the sounds.
